Export structure for FX transactions

This function enables you to define the file structure of the FX transactions generated from and aimed at being exported into an external system.

You must complete this step before exporting files, in order to ensure the smooth integration of the file data into the external system. It enables the definition of the type and location of data and potentially, conversion rules.

Accessing Setup Page for Export Structure of FX Transactions

From the pane, select the Forward Exchange Export Structure option in the folder of the module.

The setup page for the file structures of FX Transactions, is displayed with the list of set up structures in a table.

The table shows the code and description of each structure, as well as the relevant function type, the expected date format, the standard length of records in characters and the presence and absence of value equivalences in a correspondence table.

 Creating File Structures for FX Transactions
  1. In the setup page of the file structures for FX Transactions, click the Create button.
    The first page of the Structure Wizard opens up.

  1. Enter the structure's basic information.

View Fields Description

Field Description
Description Enter a description for the new structure.
Type The "Export" function is selected. No action can be performed on this field.
Length Specify the length of a record in characters including spaces. A record of the file consists of a string of characters containing the information of a single line.
Line Feed Enable this option if the entries in the file are separated by a line feed.
Date Format Select the expected date format for the transaction dates.
dd for day, MM for month and yyyy for year.
Separator Select the separator type between day, month and year of the record date.
Decimal Symbol Select the decimal separator for the amounts.

  1. Click the Set up the elements of the structure button to complete the file structure setup.

The setup page for the fields of each record, opens up.

If you click the Cancel button , the current setup is canceled and the list of the set up file structures is displayed again.

This page enables you to set up eleven fields for each record, corresponding to eleven types of information on the transaction.

  1. Specify the field settings of each entry in the file.

  1. Click the  Set up the correspondence tables button to go on with the setup.

The setup page for the correspondence tables is displayed.

  • If the codes in the file are the same as the ones in the database, click  the Test the structure button to avoid this step of the setup and go directly to the structure test.
  • To avoid the structure test and validate the setup straight away, click the  Validate button.
  • To cancel the current setup and redisplay the list of the set up file structures, click the  Cancel button.


This page consists in four tabs.

  • The Accounts tab enables you to establish equivalences between bank account codes.
  • The Transactions tab enables you to establish equivalences between the codes of transaction types.
  • The Companies tab enables you to establish equivalences between company codes.
  • The Currencies tab enables you to establish equivalences between currency codes.

In the Accounts tab, enable the option Do you want to use the equivalences set up in the Bank Accounts? if you do.

If you enable this option, the other fields in this tab are disabled.

Each tab contains a four-column table.

  • The column called N°, shows a number that identifies each equivalence.
  • The column called Internal Field, shows the bank account codes saved in the database.
  • The External from and External to columns include the bank account codes contained in the files to export.
  1. If you did not enable the option Do you want to use the equivalences set up in the Bank Accounts?, click Add to create a correspondence.

A line is added in the table.

  1. On this line, click the button in the Internal Field column to select the bank account code existing in the database.

The bank account code is displayed in the Internal Field column.

  1. In the External from column, enter the corresponding bank account code in the exported file.
  2. Click the Add button to add a new line or the Delete button to delete the selected line.

Complete the same steps again and as many times as necessary, to complete the correspondence table.

The Transactions, Companies and Currencies tabs look like the Accounts tab.

Go through the same process to define equivalences of transaction type codes, company codes and currency codes in these tabs.

  1. Click the Validate button to validate and save the structure.

The setup page for the file structures reappears with the newly saved structure at the bottom of the set up structures list.

Now the setup is completed, you can export the FX transactions. See the Data Export topic for more information.

 Duplicating File Structures

You can create structures using the copy function on an existing structure.

  1. In the setup page for Structures, select the structure to duplicate and click the Duplicate button in the actions bar.

A dialog box opens up, suggesting you modify the duplicate.

  1. Click Yes to modify the duplicate.
  2. The creation page for structures completed with the settings of the duplicated structure, opens up. An identifying number is automatically created for the structure by incrementation.
  1. In the Description area, enter the description for the new structure.

You can modify the settings of the duplicated structure.

For more information on the setup pages for File Structures, see the previous topic.

 Modifying value correspondences between Files and Database

You can enrich, modify or prune the equivalences of values between file and database at any time.

  1. In the setup page for structures, select the structure to modify and click the Modify the Equivalences button in the actions bar.
  2. The definition page for correspondence tables opens up, with the set up correspondences for the selected structure.

For more information on the fields in this page, see the topic describing the creation process for the export files structure.
