Setting up Back-office Locking

The back-office locking process is optional, but using it enables the highest level of security — particularly when the front-office management is completely dissociated from the back-office management, or when more than one user can enter and process FX Transactions in .

Using the back-office locking process has an immediate impact on the transactions entry. You need to decide whether to apply this security or not, before using .

Locking the back-office validation enables the restriction of the modification options regarding the basic and specific characteristics of each transaction. However, these elements can still be modified.

The B.O-validated transactions cannot be deleted.

When you validate a swap transaction back-office, if you want to modify it, you need to modify each leg separately, i.e. through the management pages for Spot and Forward Exchange Transactions.
Locking B.O-validated Transactions
  1. From the pane, select the FX Configuration option in the folder of the module.

The setup page for the management preferences of FX Transactions and Commercial Forecasts is displayed.

  1. In the General tab, enable the Lock B.O. Change option if you want to restrict the modification options for the B.O-validated FX transactions.
  2. Click OK to save your setup.

For more information, see the FX Configuration topic.
