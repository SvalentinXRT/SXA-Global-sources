Remittance Characteristics

You can generate remittances in two ways:

Through a single action: the batch of transfers is constituted and formatted.

Through two actions: the batch is constituted, then formatted later on. This method should be used if you are not completely sure to send all the selected transfers, or if you wait for someone else to validate the submission of the files to the Communication software.

The remittance remains editable until being formatted.

The Process by batch option of the Validation menu opens the Selection page for the display criteria of the transactions to include in the remittance.

After you validate the criteria using the OK button, the Process by batch page opens up. It displays all the validated and authorized transfers that meet your potential criteria.

The tick in the selection box of a transfer means that it is going to be processed. All transfers are selected by default.

Click Process to open the Remittance Proposal page. It groups the orders according to the applicants, profiles and constitution rules that you set up.

Three group boxes are available:

  • The Remittance(s) section displays the detailed remittances of previously selected orders.
    For every remittance, the applicant/issuer, the branch, the number, the date type (if there is a date, it means that the remittance is a single-date one), the currency management type (if there is a currency, it means it is not a multi-currency one), and the type (standard or maturity) specific to Domestic Transafers and the total amount are presented in a table.
  • The Orders section displays the orders of the remittance selected in the Remittance(s) section.
    For every order, the number, the payee, the payment currency, the amount, the execution date, the reference and the purpose are presented in a table.
  • The Processes section is a time-saving function since it enables you to launch both the constitution and the generation of the remittance, but also to start both the edition of the remittance slip for the bank, and the edition of the letter for the beneficiary.
    The Generate option (batch constitution + formatting) is selected by default. If you want to edit the remittance, you must disable this option.
Creating Remittances
  1. In the Validation menu, click the Process by batch option.

The selection page for transactions is displayed. For more information on this page, see the Selecting Transactions topic.

  1. Complete the page, if necessary, according to the transactions that you are looking for, then click OK to open the Process by batch page presenting the list of the transactions to process.
  2. If you want to remit all the validated and/or authorized orders, click the Process button; otherwise, disable the transfers that you do not want, before launching the batch processing.

The Remittance Proposal window opens up. The proposed remittance can be multi-date, single-date or mutli-currency, like in the following screenshot.

  1. Click OK to access the content of the proposed remittance.
  2. Click OK if you want to accept the proposal. The proposal window disappears and the remitted orders disappear from the Process by batch page.
Orders of generated or constituted remittances are assigned the Remitted or Prepared status and can no longer be modified. Note also that a generated remittance cannot be modified any more; you can view it only through the Remittances menu of a transaction.
