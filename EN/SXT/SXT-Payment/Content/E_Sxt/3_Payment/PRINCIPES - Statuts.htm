Statuses

Payments, Payment Requests or Transfers, are always associated with a status.

You can view all payments (Transfers, Payment Requests) at any time, by filtering them by status.

The status and its evolution tracks the actual evolution of the corresponding payment processing.

You can also organize the access rights of users according to the payment statuses.
List of the Statuses

The following table describes the various statuses and the information levels required to assign them to a transaction.

Status Transaction Type Required Information Definition
To Complete Any incomplete payment Minimum: Transaction Code, Payor, Payee, Account ID to debit, Payment Currency This default status is used when the payment request does not contain all the required elements. A payment To Complete can be modified or deleted. However, it can never be validated.
To Validate Any Transaction Type, except Payment Request the Minimum Required plus all the elements linked to the Profile and the Constitution Rule This status is automatically assigned to specify that the payment order is complete and that the user only needs to validate it. The change from the To Complete status to the To Validate status depends on the declared format.
Validated Any type of transaction for which the status To Validate   Unlike the previous status, this one requires the user's intervention.
A Validated order can be prepared and remitted to the bank. Once Validated, it cannot be modified or deleted, except if the authorized user assigns the To Change or To Delete statuses.
Authorized Any type of transaction for which the status is Validated   This optional status enables you to add a control level on the order before it is remitted.
To Change Any type of transaction with a status different from Validated, Authorized, Remitted and To Delete   This status enables you to modify the order although it has already been validated. Usually speaking, the user who validates the order is different from the user who modifies it later. An order associated with this status cannot be validated until it is modified.
To Delete Any type of transaction with a status different from Remitted   This status enables you to delete an already validated order. Usually speaking, the user who validates the order is different from the user who deletes it later.
Prepared* Any type of transaction for which the status is Validated   This status shows that the order was partially processed to be remitted to the bank. It cannot be modified any more and the remittance batch that contains it has not been generated yet.
Remitted* Any type of transaction for which the status is Prepared   This status shows that the order belongs to a batch that has already been remitted to the bank and the remittance file has already been generated.

(*) A Prepared order corresponds to a constituted remittance and a remitted order corresponds to a generated remittance.

 

For a payment order or request with the To Complete status, the later statuses can be assigned only when the required information - specific to the Payment Type, to the Profile and to the Constitution Rule - are entered.
Modifying Statuses Manually

When you perform an action on a payment order, the corresponding status is automatically associated with the order.

However, you can change the status manually, if the payment order complies with the assignment specifications for the new status (see the table above).

This modification can be performed from different menus of the module.

The statuses that can be assigned by manuam modification, are the following:

  • Validated
  • Authorized
  • To Delete
  • To Change
  • Remitted

The Authorized status has the same features as the Validated status, but it is subsequent to Validated. The activation of the optional command that enables the authorization of the payment order, depends on the control level decided by your company.

The manual modification of statuses, i.e. using a specific command, is possible only for complete orders.
