Transfers Import

This function enables you to import transfers from a flat file containing the transfers created within an external system.

Before the import, make sure that you set up the import structure properly in the module. If the file does not match the defined structure, the import generates errors. See the section Structure for Exchanged Files, page 1topic Structure for Exchanged Files for more information on the setup of structures for files to import.

You cannot import the same file twice. You can recycle the generated rejects (in case of irrelevant information in a record) via the management function of the various payment types.
Accessing Setup Page for Import Tasks

In the pane, select the Import Payment Types option in the folder of the module.

The page of file import tasks is displayed with the list of the tasks saved in a table.

The table shows the task number and description, the file location and the associated file structure.

When this page displays a list of already created tasks, you can import your payments in three different ways through the Process dropdown menu:

  • Import: the import is launched straight away.
  • Run Import Background: the import is launched straight away, but you can continue working in the application.
  • Run Import Remote: the import is launched before or after your usage time.

If this page does not display any created tasks, then you have a fourth option: the import can also be done when you create the task through the wizard.

 Creating and Activating Import Tasks
  1. In the setup page for Import Tasks, click the Create button.

The first page of the import wizard opens up.

A sequential number is automatically created for the import task and displayed in the field You are going to create the task number.

  1. Enter a description for the import task in the Description field.
  2. In the File field, click the button to view the browser and select a file to test.

A contextual menu appears.

  1. In the browser, select the directory where the file is located.
    The path appears in the Selected Path field and the files of the directory are listed in the table at the bottom of the window.
  2. Double-click the name of the file to select it, or click the button to validate the selection.

The wizard appears again and you can view the file path in the File field.

  1. In the Structure area, click the button to display the list of set structures. In the contextual window, double-click the line to select it.
  2. Click the button  Setting up the import options to go on with the setup.

The setup page for Options is displayed.

This page enables you to define the import settings, such as the bank format type.

If you want to:

  • go to the next step and save the task just as it is, without launching it, click the Validate button,
  • cancel the current setup and redisplay the list of the export tasks, click the Cancel button.
  1. Click the Import button to launch the import task.

If you want to:

  • save the task just as it is, without launching it, click the Validate button,
  • cancel the current setup and redisplay the list of the export tasks, click the Cancel button.

The import is launched and a new page is displayed showing the task progress.

  1. Once the task is completed, click the Close button at the bottom of the page.

The page listing the import tasks is redisplayed, the newly saved tasked appears at the bottom of the list, then the import report opens up.

You can print, export this report or send an electronic notification of it.

If the Rejected field contains a number different from 0, rejects have been generated during the import. For more information, see the following topicsection, Transfer Rejects.Transfer Rejects, from page 1.

Once the payments are imported, you can:

  • modify them via the management function for Payment Requests or Transfers (for more information, see the topicProcessing Paymentssection Processing Payments, from page 1),
  • associate them with remittances to transmit to the bank (for more information, see the topic Remittances Constitutionsection Remittances Constitution, from page 1).
