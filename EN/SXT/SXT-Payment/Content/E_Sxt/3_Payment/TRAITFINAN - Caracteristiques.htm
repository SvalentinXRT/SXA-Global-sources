Payment Characteristics

As specified in the Payments topic, the term Payment refers to the payment requests plus the various transfer types.

Remember that any payment that is not a Payment Request is necessarily a Transfer.

Payment Requests

Any incomplete order (missing elements compared with the requirements of each transfer type) is a Payment Request.

All the payment requests already entered or integrated are grouped in the Payment Requests link of the Transactions menu.

The Payment Requests link provides different functions enabling you to:

  • create, modify, delete or view Payment Requests,
  • validate, authorize or assign To Change or To Delete statuses, once Payment Requests do not bear the To Complete status any longer.
  • get reports on the payment characteristics via the Documents button and selecting the Ticket command.
The Validate, Authorize, Modification Request and Deletion Request options of the Process button do not work for Payment Requests, since their status is To Complete.

When a payment request has not been integrated, you need to create it in the module, through the Payment Requests link of the Transactions menu.

Entering Payment Requests
  1. From the homepage of the module, click the Payment Requests link of the Transactions menu.

The list of the Payment Requests already integrated to the module, is displayed.

  1. Click the button.
  2. Select your creation mode for the new payment request: from an existing template or not, then click OK.

For more information on how to create Transactions from an existing template, see the Creating Templates topic.

  1. In the Payment Request - Create page, enter the following mandatory elements:
  • Transaction Code
  • Payor (Participants tab)
  • Payee (Participants tab)
  • Account ID to debit (Participants tab)
  • Transaction Curreny (Definition tab)
  • Transaction Amount (Definition tab)
  1. Click OK to validate this payment request creation and go back to the list of the integrated payment requests, or click Apply to validate and create another one.

Payment Requests can be modified for two main reasons:

  • To complete some missing information or correct errors in this information.
  • To turn the status into To Validate, by completing the mandatory information specific to the transfer selected during creation.
 Modifying Payment Requests
  1. From the homepage of the module, click the Payment Requests link of the Transactions menu.

The list of the Payment Requests already integrated to the module, is displayed.

  1. Select the payment request that you want to edit, then click Change to open the Payment Requests - Change page.
  2. Edit the relevant elements, then click OK to save your actions and go back to the Payment Requests list.
 Deleting Payment Requests
  1. Select the requests that you want to delete from the payment request list.
  2. Click or press the Delete key.
  3. Confirm your action by clicking Yes in the confirmation dialog box.
 Domestic Transfers

Domestic transfers are defined as national funds transfers made in the national currency.

The Domestic Transfers link in the Transactions menu displays the list of the integrated domestic transfers.

Most of the main information on transfers can be viewed from this page.

The Create button in the Domestic Transfers contextual menu triggers the Domestic Transfers - Create, after the potential selection of a template.

The upper part of the page includes four mandatory elements:

  • Transaction Code: This code is mandatory for domestic transfers.
  • The Transaction Reference. It is required for the order to be validated.
  • The Transaction Number. This number is automatically generated, using the following formula: company code (3 characters) + transaction code (4 characters) + counter (6 positions).
  • The Transaction Status. The default status is "To Complete", until the order is fully completed.

The lower part consists of four entry or selection tabs plus one follow-up tab:

  • The Participants tab deals with the transfer issuer and beneficiary.
  • The Definition tab enables the specification of the transaction.
  • The Instructions tab contains information on how to process the transaction.
  • The Back-office tab recaps the life cycle of the payment order. It is automatically completed.
  • The Declaration tab is designed to complete the declaration required by the national Central Bank.
Creating a Domestic Transfer
  1. From the homepage of the module, click the Domestic Transfers link of the Transactions menu.

The list of the domestic transfers already integrated to the module, is displayed.

  1. Click the button.
  2. Select the creation mode to use for the new domestic transfer: from an existing template or not, then click OK.

For more information on how to create Transactions from an existing template, see the Creating Templates topic.

  1. Complete the fields with the information in your possession, i.e. :
  • Applicant (Participants tab)
  • Transaction Code
  • Reference
  • Payor (Participants tab)
  • Payee (Participants tab)
  • Account ID to debit (Participants tab)
  • Currency (Definition tab)
  • Amount (Definition tab)
  1. When the characteristics of the transfer are completed, and if you want to re-use this information for other transfers, click Save As Template... and enter a code and a description in the displayed Save As Template... page.
  2. Click Apply to save your entries and go on with the creation of other transfers, or OK to save your entries and go back to the list of the integrated transfers.

If some elements required for proper transfers are missing – according o the format – a dialog box is displayed.

It specifies the missing parts. You have two options:

  • Click Yes to create the order notwithstanding the warning, and accepting the To Complete status.
  • Click No to complete the entry form so that the order can take the To Validate status.

The dialog box continues to appear, after you save the order, until the mandatory elements of this order are all gathered.

Whatever their types, transfers can be modified, provided they have not yet been validated. Afterwards, they can still be viewed, unless a user assigned them the To Change status. In this case, the transfer characteristics can be modified.
Modifying a Domestic Transfer
  1. From the homepage of the module, click the Domestic Transfers link of the Transactions menu.

The list of the domestic transfers already integrated to the module, is displayed.

  1. Select the transfer that you want to edit, then click Change to open the Domestic Transfers - Change page.
  2. Edit the relevant elements, then click OK to save your actions and go back to the domestic transfer list.
 Deleting a Domestic Transfer
  1. Select the debit that you want to delete from the integrated domestic transfer list.
  2. Click or press the Delete key.
  3. Confirm your action by clicking Yes in the confirmation dialog box.
Important Only the transfers bearing the To Complete, To Change and To Delete statuses can be deleted.

 Commercial Transfers

Commercial transfers are encashable domestic transfers. The client's bank offers the provider either payments at maturity or payments in advance with interests.

Commercial transfers are carried out in Euros only.

Two formats are available:

  • EDIFACT (international standard)
  • VCOM400 (French standard)
The module enables you to import commercial payments, invoice lines included. To be imported, transfer files must comply to the file structure defined for , and transaction codes, accounts, applicants, third parties, etc. need to be previously set up.

The Commercial Transfers link in the Transactions menu displays the list of commercial transfers, either imported or manually entered.

Most of the main information on transfers can be viewed from this page. 

The Create button in the Commercial Transfers contextual menu triggers the Commercial Transfers - Create page, after the potential selection of a template.

The Invoices tab enables to enter and manage the invoice lines required to complete commercial transfers.

Creating a Commercial Transfer
  1. From the homepage of the module, click Commercial Transfers in the Transactions menu.

The list of imported commercial transfers is displayed.

  1. Click the button.
  2. Select the creation mode to use for the new commercial transfer: from an existing template or not, then click OK.

For more information on how to create Transactions from an existing template, see the Creating Templates topic.

  1. Complete the fields with the information in your possession, i.e.:
  • Transaction Code
  • Reference
  • Applicant (Participants tab)
  • Payor (Participants tab)
  • Payee (Participants tab)
  • Account ID to debit (Participants tab)
  • Beneficiary (Participants tab)
  • Allocation of Charges (Participants tab)
  • Currency (Definition tab)
  • Amount (Definition tab)
  1. When the characteristics of the transfer are completed, and if you want to re-use this information for other transfers, click Save As Template... and enter a code and a description in the displayed Save As Template... page.
  2. Click Apply to save your entries and go on with the creation of other transfers, or OK to save your entries and go back to the list of imported commercial transfers.
A Commercial Transfer remains with the status To Complete until an invoice line is linked to the transfer. If trying to fill in the Amount field without any invoice line previously imported through the Invoice tab, an error message will be displayed, specifying that the indicated amount is incorrect.
Entering an Invoice Line
  1. Click the Invoices tab from a selected commercial transfer or a commercial transfer being created.
  2. Click Create to open the creation page for Invoice Lines.
  3. The Expenses option is automatically selected. If your invoice includes a credit note, select the Receipt option.
  4. Select an invoice type.
  5. Assign it an internal reference as well as a reference to the Beneficiary.
  6. In the second group box, enter the amount in the Amount Paid field. The invoiced amount is automatically calculated.
  7. Click Apply to save the invoice line and enter another one or OK to save the invoice line and go back to the Invoices tab of the commercial transfer.
Before creating an invoice line, you need to define the transaction currency in the Definition tab.
Modifying an Invoice Line
  1. Click the Invoices tab from a selected commercial transfer or a commercial transfer being created.
  2. Select the invoice line to edit and click Change.
  3. Enter the modifications.
  4. Click OK to save your entry.
 Deleting an Invoice Line
  1. Click the Invoices tab from a selected commercial transfer or a commercial transfer being created.
  2. Select the invoice line to edit and click Delete.
  3. Click Yes in the dialog box to confirm the deletion.
 Modifying a Commercial Transfer
  1. From the homepage of the module, click Commercial Transfers in the Transactions menu.

The list of imported commercial transfers is displayed.

  1. Select the transfer that you want to edit, then click Change to open the Commercial Transfers - Change page.
  2. Edit the relevant elements, then click OK to save your actions and go back to the commercial transfers list.
 Deleting a Commercial Transfer
  1. Select the transfer that you want to delete from the list of imported commercial transfers.
  2. Click or press the Delete key.
  3. Confirm the deletion of the commercial transfer by clicking Yes in the dialog box.
