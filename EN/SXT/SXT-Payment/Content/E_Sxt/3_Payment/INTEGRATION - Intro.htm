Importing External Data

This sectiontopic presents the import of Transfers in the module. You can then use the transfers created in another system to avoid a double entry.

You first need to:

  • export in a flat file and from the external system the transfers created in this system,
  • set up the import structure in the module.

The import consists in integrating into the database a flat file exported from the external system.

The import process can generate rejects: the rejected transactions are not imported, but they are kept in the history where you can correct them, and relaunch their import into .

Once the transfers are imported, you can:

  • modify them using the transfers management function,
  • link them to remittances to transmit to the bank.

You can also create your transfers manually. For more information on transfer creation and modification, see the Transfers topic.section Transfers, page 1.
