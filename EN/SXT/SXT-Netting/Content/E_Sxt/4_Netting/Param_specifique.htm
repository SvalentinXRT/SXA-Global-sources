Specific Setup

Before using , you need to set up some elements in the Setup menu, from the Modules Tasks pane.

Click the Setup task to display the full list (six elements).

Posting Flows

The compensation transactions can be viewed from the other modules.

The setup enables you to choose the way the compensation transactions from a netting chart are displayed into Payment and Cash.

You may select posting conditions from the list or create them from scratch.

Configuration

Click Configuration to open the setup page called Configuration - Change, which consists of two tabs:

  • Invoices
  • Netting Process

The Invoices tab enables you to define the compensation's calculation basis and the processing date.

Select your option.

The Netting Process tab enables the insertion of non-business days into the netting cycles, the deletion of the closed simulations and the application of controls over the posting process.

Select your option.

 Netting Chart

The Netting Center is the core of the module as it initiates and manages the netting process. Participants are linked to the Center and submit their data so that it can proceed with the netting process and calculate the corresponding payments. Participants' access rights must be specified to prevent unauthorized users from any input, modification or deletion of invoices.

The main characteristics of the netting chart are to be set up through the definition of the five tabs in the dialog box:

  • General which identifies the chart
  • Internal Entities which specifies the internal partakers linked to a same netting center
  • External Entities which specifies the external partakers of the netting chart
  • Invoicing which defines the currencies authorized for the netting chart
  • Netting Transactions which defines the default transaction code and applicant used for the chart
Only one netting center may be set up in the module.

Netting Rates

The netting process is based on two currency conversions:

  • Invoicing currency into netting currency
  • Netting currency into payment currency

You can use one of these rates for the process:

  • Market Rate (Average and Ask/Bid)
  • Budget Rate

You can also set up customized rates for the calculation of each conversion.

Cycles

Cycles are key elements to . They define the deadlines for each step of the netting process.

One cycle consists of four steps:

  • Invoices input
  • Netting simulation
  • Netting validation
  • Settlement

The setup of cycles is quite user-friendly, it can be manual or automatic.

Watch video

Cycle Structure

Once you created a cycle, you may set up its internal structure and define its steps.

You need to specify the calendar, frequency and steps limits for the cycle structure.

You can also define a cycle structure to be used as a template for later cycle creation. This template is general: it is not related to any chart.
