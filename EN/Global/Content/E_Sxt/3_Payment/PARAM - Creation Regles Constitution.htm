Creating Constitution Rules

During the remittance generation, Payment uses Profiles to identify the bank formats and protocols used, and then Constitution Rules to adapt profiles to the agreements negotiated with banks.

For example, the MT101 format which is among the most flexible formats, can be restricted by a bank. The constitution rule will define the restriction levels of the original format.

Setting up Constitution Rules
  1. To access this function, in the pane, select the Payment Elements option in the folder of the module.

The setup page for Receipts Elements is displayed. Click Constitution Rules.

The list of the already set up rules, is displayed. You can delete or modify an existing rule, or create another one. To create a constitution rule, click Create.

  1. Complete at least the Transaction Code field in the Criteria group box.
  2. In the Processes group box, select the options that you want to appear on the remittance:
    • Multi-currency: these remittances contain orders expressed in different currencies. You can also select the Alert option, if you want to be notified every time there is a multi-currency remittance.
    • Multi-date: these remittances contain orders with different maturity dates or execution dates. You can also select the Alert option, if you want to be notified every time there is a multi-date remittance.
    • Multi-Remit.: every remittance contains a grouped orders batch, whether the orders are multi-currency or multi-date, or not.
    • Limit: the number of orders per remittance is limited. Then, enter this number in the Max. Orders/Remittance field.
    • Debit remittance: select from the dropdown list a debit type for your remittance. Unit Fee: every order is stated. Global: there are no details, only the global sum appears. Global by Currency: the global sum of the remittance is presented by currency, with a global sum for every currency.
  3. You can enter comments in the Description (maximum 35 alphanumerical characters) and Next fields.
  4. Once you completed the mandatory fields, click Apply to validate your action and go on with the profiles creation, or click OK to save the new profile and go back to the profile list.

To specify an exception, you also need to complete the following fields before creating the profile:

  • Applicant Branch: if the branch does not operate in the same way as the branches of the general rule.
  • Applicant: if you want to specify a particular applicant.
 Modifying a Constitution Rule
  1. Select the Payment Elements option in the  folder of the module. The setup page for Receipts Elements is displayed. Click Constitution Rules.
  2. Select from the list the rule you want to modify.
  3. Click Change. The detailed page for the selected rule is displayed.
  4. Edit the relevant elements, then click OK to save your actions.
 Deleting a Constitution Rule
  1. Select the Payment Elements option in the  folder of the module. The setup page for Receipts Elements is displayed. Click Constitution Rules.
  2. Select from the list the rule you want to delete.
  3. Click Delete to delete the selected rule.
  4. Confirm your action by clicking Yes in the confirmation dialog box.
You should create one constitution rule per transfer type: Domestic Transfer, Commercial Transfer, International Transfer and Treasury Transfer.
