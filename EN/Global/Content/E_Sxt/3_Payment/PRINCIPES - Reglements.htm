Payments

Payments is a generic term that refers to both Payment Requests and Orders, whether they are complete, incomplete, waiting for transmission or remitted.

In the module, the Transactions function provides a single access to all transactions.

From this menu, you can also access the Payment Requests by clicking Payment Requests.

The Transactions menu does not give access to the details of payments. It only displays the minimum of mandatory data for the integration of the payment orders or requests from a G/L file.

Requests become Orders as soon as their types are recognized. These orders are then progressing to validated, authorized etc. transfers.

The various Transaction Categories are identified by codes which are created from the Common Elements menu and the Transaction Codes sub-menu. Every Transaction Code is associated with one Transaction Category that can be Domestic Transfer, International Transfer, Treasury Transfer, etc.
Minimum Information Required

Payments must always contain a minimum of required information:

  • Transaction Type (International Transfer, Domestic Transfer, Commercial Transfer, Treasury Transfer, etc.)
  • Payor
  • Payee
  • Account ID to debit
  • Currency of the payment

The default planned payment date is also mandatory. If it is not the right one, you need to modify it.

Any Payment Request with missing elements is rejected during integration. The (third party) account to credit is automatically displayed.
Turning Payment Requests into Transfers

When you want to create an order in the module - by integration or entry - and when some required data are missing, you can make a payment request (containing the minimum required data) that you can view and complete by clicking Payment Requests.

To turn your payment request into a transfer, you need to provide the following information:

  • Transaction Amount
  • mandatory information specific to each Transaction Category
