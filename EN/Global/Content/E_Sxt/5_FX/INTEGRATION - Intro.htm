Importing External Data

This topic presents the import of commercial transactions in . You can then use again the commercial transactions created in another system to avoid a double entry.

You first need to:

  • export in a flat file and from the external system, the commercial transactions created in this system,
  • set up the structure of the files to import, in .

The import consists in integrating into the database a flat file exported from the external system.

The import process can generate rejects: the rejected transactions are not imported, but they are kept in the history where you can correct them, and relaunch their import into .

Once the commercial transactions are imported, you can:

  • modify them via the management function for Commercial Transactions,
  • associate them with FX Transactions.

You can also create your commercial transactions manually. For more information on the creation and modification of Commercial Transactions, see the Commercial Transactions topic.
