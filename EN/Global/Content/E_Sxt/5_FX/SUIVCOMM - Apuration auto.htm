Automatic Update

Automatic Update is performed according to the management mode selected for Commercial Transactions. For more information on the impact of this setup on Update, see the Management Methods for Commercial Transactions topic.

To enable the automatic update, Commercial Transactions must meet the following requirements.

  • The type of the actual flow in the Treasury must be export receipt or import disbursement.
  • With the Global, Decrementation Mode, the transaction currency of the forecast must be the same as the transaction currency of the bank actual, otherwise the update is automatic.
  • With the Global, Budget Decrementation Mode, the budget code of the forecast must be the same as for the bank actual.
  • The company for the forecast must be the same as for the bank actual.
  • The status of the commercial transaction must be different from the "Settled" status.

The forecasts which were not automatically updated, must be manually updated or manually deleted.

Automatic Update in Global, Decrementation Mode or Global, Budget Decrementation Mode
  • Commercial Transactions are managed globally during the bank reconcilation.
  • The decrementation is based on the FIFO concept: the non-updated transactions and the oldest ones in the month corresponding to the delivery date are updated first with the actual flows from the module.
  • The actual flows with the Export Receipt type enable the update of the Receipt Forecasts whereas the actual flows with the Import Disbursement type enable the update of the Disbursement Forecasts.
  • Here are the criteria used for Reconciliation: flow type, company, transaction currency, month of the delivery date for the commercial forecast, month of the transaction date for the actual flow, budget code.
  • Once Reconciliation is completed, the amount in position for Commercial Transactions is updated according to the reconciled amount, and the transaction status reaches "Settled" or "Updated" depending on whether the amount in position is null or not.

The forecasts which were not automatically updated, must be manually updated.

 Automatic Update in Detailed, Reconciliation Mode
  • Commercial Transactions are managed individually during Reconciliation, transaction by transaction and amount by amount.
  • Once Reconciliation is completed, the commercial transactions amount is set to zero and the transaction reaches the "Settled" status.

The forecasts which were not automatically updated, must be manually deleted.
