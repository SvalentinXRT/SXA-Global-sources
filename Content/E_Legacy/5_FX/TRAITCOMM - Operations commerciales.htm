Opérations commerciales

Cette rubrique concerne la gestion des opérations commerciales. Les opérations commerciales sont traitées en fonction du mode de gestion configuré. Pour plus d'informations sur la configuration du mode de gestion des opérations commerciales, consultez la rubrique Configuration FX.

Les opérations commerciales sont identifiées par un statut de réalisation. Lorsqu'elles ne sont pas réalisées, elles ont le statut de prévisions. Elles peuvent être créées par importation de fichier ou saisies manuellement dans le module .

  • L'importation est effectuée avec un fichier plat généré depuis un système externe et contenant toutes les informations des opérations commerciales (pour plus d'informations, consultez la rubrique Importation des prévisions commerciales).
  • Consultez les procédures ci-après pour plus d'informations sur la saisie manuelle des opérations.

En mode saisie détaillée avec rapprochement, les flux correspondants sont postés dans la trésorerie avec le statut « prévision confirmée » dès qu'une prévision commerciale est créée ou modifiée.

Pour plus d'informations sur le mode saisie détaillée avec rapprochement, consultez la rubrique Modes de gestion des opérations commerciales.

Les opérations commerciales ont le sens recette à l'export et le sens dépense à l'import.

Les prévisions commerciales doivent être créées en amont des transactions de change à terme. Une fois les transactions de change à terme créées, vous pouvez lier celles-ci aux prévisions commerciales correspondantes.

Une opération commerciale peut avoir trois statuts : ouverte, apurée partiellement (opération avec paiements partiels) et fermée (opération apurée totalement).

Pour des informations sur le suivi des opérations commerciales, consultez la rubrique Configuration FX.

Accéder à la page de gestion des opérations commerciales
  1. Dans le volet , sélectionnez l'option Prévisions dans le dossier du module .

La page de modification du filtre défini par défaut pour cette fonctionnalité s'affiche.

Cette page vous permet de définir des critères de filtrage permettant de restreindre ou d'élargir la sélection des opérations à afficher dans la page des opérations commerciales. Si aucun critère n'est renseigné dans cette page, toutes les opérations commerciales existantes seront affichées dans la page. S'il existe un grand nombre d'opérations, restreindre la sélection en renseignant des critères vous permet de gagner en lisibilité.

  1. Configurez le filtre, puis cliquez sur OK.
Pour une description des champs de cette page, consultez la rubrique ci-après qui décrit la création de filtre.

La page de gestion des opérations commerciales s'affiche avec la liste des opérations créées sous forme de tableau.

Ce tableau affiche la liste des opérations commerciales avec le type d'opération, le numéro de l'opération, les dates d'opération et de livraison, la société, le montant de l'opération, sa contre-valeur, le montant non couvert, un indicateur de couverture modifiée, les cours de change budgétaire et marché, la nature budgétaire, le compte bancaire lié, la tierce partie, le statut de l'opération (ouverte, dénouée, apurée partiellement), son origine (interne ou externe), les numéros de référence et de rattachement le cas échéant, ainsi que les remarques éventuelles.

Recettes/Dépenses

Définissez le sens de l'opération en activant la case d'option Recettes ou la case d'option Dépenses.

Transaction

Ce champ vous permet de sélectionner un type de transaction en fonction du sens d'opération choisi.

Cliquez sur le bouton pour afficher la liste des types de transaction existant pour le sens d'opération choisi puis, dans la fenêtre contextuelle qui s'affiche, double-cliquez sur une ligne pour la sélectionner.

Le code et la description du type de transaction sélectionné s'affichent dans le champ Transaction.
N°

Ce champ, non modifiable, affiche l'identifiant de l'opération.

Vous ne pouvez pas saisir d'informations dans ce champ.

Il est à blanc lors de la création, puis automatiquement renseigné au moment de l'enregistrement de l'opération.

    
N° rattach.

Ce champ facultatif vous permet de regrouper des prévisions commerciales et des transactions de change sous un numéro de rattachement commun. Ce numéro vous permet de déterminer l'affectation des couvertures relevant d'un type d'activité, d'une zone géographique, d'un projet, etc. Il peut être utilisé ultérieurement comme critère de traitement comptable ou de filtrage. Pour plus d'informations sur les numéros de rattachement, consultez la documentation du module .

Pour sélectionner un numéro de rattachement, utilisez le bouton comme expliqué précédemment.

    
Statut

Ce champ, non modifiable, affiche le statut de dénouement de l'opération : ouverte ou fermée.

Zone Général
Société

Ce champ vous permet de sélectionner la société associée à l'opération.
Pour sélectionner une société, utilisez le bouton comme décrit précédemment.

La devise de contre-valeur, le cours budget et le cours d'opportunité sont automatiquement renseignés à partir de la société sélectionnée.
Montant

Les deux champs affichent la devise principale et le montant de l'opération exprimé dans la devise principale.

Sélectionnez la devise principale en utilisant le bouton  comme décrit précédemment. Le code de la devise s'affiche dans le premier champ.

Dans le champ de saisie situé après la devise, saisissez le montant de l'opération exprimé dans la devise principale ou calculez le montant en utilisant la calculatrice. Pour plus d'informations sur l'utilisation de la calculatrice, cliquez ici.

Une fois le montant et la devise principale de l'opération renseignés, le montant en position et le montant de la contre-valeur sont automatiquement calculés et affichés.

Si la société a également été renseignée, les valeurs des cours de change budgétaire et d'opportunité sont alors automatiquement alimentées à partir de la devise de l'opération et de la société.
En position

Ce champ affiche la position du solde de l'opération après apuration partielle ou totale.

Cette zone, renseignée automatiquement à partir du montant de l'opération, n'est pas modifiable. A chaque apuration partielle, la position est décrémentée du montant du paiement.

Pour plus d'informations sur l'apuration, consultez la rubrique Apuration.

    
Contre-valeur

Les deux champs affichent la devise de contre-valeur et le montant de l'opération exprimé dans la devise de contre-valeur.

Le montant de la contre-valeur est automatiquement renseigné à partir du cours de change budgétaire et du montant de l'opération.

La devise de contre-valeur proposée par défaut correspond à la monnaie locale de la société associée à l'opération.

   (devise)
Date TRN

Ce champ affiche la date à laquelle est effectuée l'opération commerciale. Il est défini par défaut à la date du jour.

Pour modifier cette date, saisissez une date dans le champ ou utilisez le calendrier. Pour plus d'informations sur l'utilisation du calendrier, cliquez ici.

  
Date livraison

Ce champ affiche la date d'échéance de l'opération commerciale. Cette date est positionnée par défaut à la date du jour.

Cette date peut être remplacée par une date identique ou ultérieure à la date de l'opération.

Pour modifier cette valeur, saisissez une date dans le champ ou utilisez le calendrier.

  

Zone Cours

Cette zone affiche la valeur des cours de change budgétaire et d'opportunité.
Cours budget

Ce champ affiche le cours de change budgétaire des devises de la transaction. Ce champ permet de budgétiser les prévisions commerciales.

Lorsque les devises de la société et de l'opération ont été renseignées, ce champ est alimenté à partir du paramétrage effectué dans le module . Cette valeur est modifiable. C'est le cours budgétaire du mois de la livraison qui est affiché dans la zone et non le cours budgétaire du mois de la date d'opération.

Si aucun cours de change budgétaire n'a été défini pour le couple de devises et la période sélectionnés, vous devez saisir une valeur dans ce champ afin que le montant de la contre-valeur puisse être calculé.

    
Cours d'opportunité

Ce champ affiche le cours de change de marché des devises de l'opération commerciale. Ce champ permet de comparer la performance d'une stratégie de couverture par rapport à une stratégie de non-couverture.

Cette valeur est alimentée à partir du paramétrage effectué dans le module . Elle est modifiable.

C'est le cours à terme de l'échéance de l'opération, à la date d'opération, qui est affiché dans ce champ.

    
Compte

Ce champ affiche le compte bancaire associé à l'opération commerciale.

Ce champ n'est affiché qu'en mode saisie détaillée avec rapprochement.

Les zones Comptes et Code flux indiquent que les opérations commerciales saisies dans le module sont positionnées dans le module en tant que prévisions sur ces comptes et avec les mêmes codes.

  
Code flux

Ce champ affiche la nature du flux de trésorerie rattaché à l'opération.

Il n'est affiché qu'en mode saisie détaillée avec rapprochement.

  
Conversion

Cette fonction vous permet de calculer la contre-valeur du montant de l'opération exprimée dans la devise du compte bancaire.

Ce bouton n'est affiché qu'en mode saisie détaillée avec rapprochement.

    
Montant de conversion

Ce champ affiche la contre-valeur du montant de l'opération exprimée dans la devise du compte bancaire.

Ce champ n'est affiché qu'en mode saisie détaillée avec rapprochement.

    
Code budgétaire

Ce champ affiche la nature budgétaire associée à l'opération commerciale.

Si le mode saisie globale avec décrémentation budgétaire a été choisi, cette information est obligatoire car elle constitue un critère de rapprochement.

    
Tiers Ce champ vous permet de sélectionner la tierce partie de l'opération.    

Cliquez sur le bouton pour changer le sens d'expression du cours des valeurs de la zone Cours (cours de change budgétaire, cours de change d'opportunité). Les valeurs affichées correspondent alors au sens devise de contre-valeur contre devise principale, mais le sens de la transaction ne change pas.

Cliquez à nouveau sur le bouton pour réafficher les valeurs dans le sens de la transaction.

  1. Cliquez sur l'onglet Zones utilisateur.

Cet onglet vous permet de définir, sous forme de texte libre, des renseignements complémentaires sur la transaction.

  1. Dans le premier champ de saisie, saisissez les commentaires que vous souhaitez inclure dans le journal.
  2. Dans le champ Référence, saisissez, le cas échéant, le numéro de référence externe de l'opération. Ce numéro constitue également un critère de recherche et de sélection.
  3. Cliquez sur le bouton OK pour valider la création de l'opération commerciale ou sur le bouton Annuler pour annuler.

 Convertir le montant dans la devise du compte bancaire

La page de conversion des montants s'affiche lorsque vous cliquez sur le bouton Conversion situé dans l'onglet Caractéristiques de la page de détail d'une opération commerciale.

Le montant de l'opération exprimé dans la devise de l'opération est affiché dans le champ Montant.

Le champ Cours affiche le cours de change de la devise du compte bancaire contre la devise de l'opération. Le couple de devises est indiqué avant la valeur du cours de change.

La devise du compte bancaire est affichée dans le champ Contre-valeur.

La devise de l'opération est affichée dans le champ Montant.

Saisissez la valeur du cours de change pour ce couple de devises dans le champ Cours.

Le montant de la contre-valeur est automatiquement calculé et affiché dans le champ Contre-valeur.

Cliquez sur OK pour valider le calcul.

La page de détail de l'opération commerciale est réaffichée avec le montant de la contre-valeur calculé.
