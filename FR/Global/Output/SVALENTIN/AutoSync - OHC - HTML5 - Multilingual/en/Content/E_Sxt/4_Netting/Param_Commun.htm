Common Setup

Sage XRT Treasury platform uses elements and data from the market. They are defined in Common Setup and apply to all the modules of Sage XRT Treasury.

Calendars

Setting up Calendars enables the definition of closing days and bank holidays for companies. In Netting, these calendars are used to set up Cycles and Cycle Structures.

Currency

Setting up Currencies enables the creation of a default reference currency used for all transaction conversions, as well as the definition of currencies for each currency account.

Country

Setting up Countries is required for the Country Codes associated with Companies and Third Parties.

Company

Setting up Companies is required for their association with the participants in the netting process. 

Bank

Setting up Banks enables their association with accounts and then companies. Codes and descriptions for banks must be defined.

Branch

Setting up Branches enables the distinction between regular and issuing branches during the transaction input into Netting.

Account

The Account setup is based on the Company and Bank setups. It creates a link between the two types of entity: banks and companies.

Transactions

This setup is essential to the Netting module. It consists of the creation and the identification of transaction codes.

Economic Reason

Setting up Economic Reason Codes enables the identification of the various transaction types. In Netting, these codes are then used during the invoice creation to keep a track of the process.

Economic Types

In Netting, this setup enables checking processes during payment posting process.

Link Numbers

This setup is useful for all users of all Sage XRT Treasury products.

In Netting, this link number is optional and enables invoices to be linked together. Invoices with the same link number can then be grouped, for reporting purposes, for example.
