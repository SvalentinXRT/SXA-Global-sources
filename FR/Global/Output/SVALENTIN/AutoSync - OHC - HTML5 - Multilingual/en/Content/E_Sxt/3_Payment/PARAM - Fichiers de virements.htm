Transfer Files

The integration of your payment requests and/or orders originating from your G/L system is a precious and time-saving tool.

This function enables you to define the file structure for the transfer files generated from an external system and imported in the Payment module.

You must complete this step before exporting files, in order to ensure the smooth integration of the file data into the database. It enables the definition of the type and location of data and potentially, conversion rules.

Accessing the Setup Page for File Structures of Transfer Files

To access this function, in the Dashboards Tasks pane, select the Payment Elements option in the Setup folder of the Payment module.

The setup page for Receipts Elements is displayed. For example, click the Domestic Transfers Import Structure link.

The setup page for Domestic Transfers Import Structure is displayed with the list of set up structures in the form of a table.

The table shows the code and description of each structure, as well as the relevant function type, the expected date format, the standard length of records in characters and the presence and absence of value equivalences in a correspondence table.

Creating a Transfer File Structure
  1. In the setup page for Domestic Transfers Import Structure, click the Create button.
    The first page of the Structure Wizard opens up.

  1. Enter the structure's basic information, as described below.

View Fields Description

Field Description
Description Enter a description for the new structure.
Type The "Import" function is selected. No action can be performed on this field.
Length Specify the length of a record in characters including spaces. A record of the file consists of a string of characters containing the information of a single line.
Line Feed Enable this option if the entries in the file are separated by a line feed.
Date Format Select the expected date format for the transaction dates.
dd for day, MM for month and yyyy for year.
Separator Select the separator type between day, month and year of the record date.
Decimal Symbol Select the decimal separator for the amounts.
Nb of digits after decimal Select the number of digits after decimal.
  1. Click the Set up the elements of the structure button to complete the file structure setup.

The setup page for the fields of each record, opens up.

If you click the Cancel button , the current setup is canceled and the list of the set up file structures is displayed again.

This page enables you to set up several fields for each record, corresponding to the same number of information types.

Now the setup is completed, you can import the domestic transfers files.

Duplicating File Structures

You can create structures using the copy function on an existing structure.

  1. In the setup page for Structures, select the structure to duplicate and click the Duplicate button in the actions toolbar.

A dialog box opens up, suggesting you modify the duplicate.

  1. Click Yes to modify the duplicate.
  2. The creation page for the structure completed with the settings of the duplicated structure, opens up. An identifying number is automatically created for the structure by incrementation.
  1. In the Description area, enter the description for the new structure.

You can modify the settings of the duplicated structure.

For more information on the setup pages for File Structures, see the previous sectiontopic.

Modifying value correspondences between Files and Database

You can enrich or modify the correspondences of values between file and database at any time.

  1. In the setup page for Structures, select the structure to modify and click the Modify the Equivalences button in the actions toolbar.
  2. The definition page for Correspondence Tables opens up, with the set up correspondences for the selected structure.

For more information on these fields, see the sectiontopic which describes the creation process for the structure of files to import.

The transaction codes mentioned in the files must appear in the files integrated into Payment. All the records for which the transaction code is not known, are rejected. This rule is applied on every parameterizable element in the product and in the relevant Sage XRT Treasury modules (Account Codes, Account IDs, Payor Code, Payee Codes, Branch Code...).
