Economic Types

Economic Types are only used for the statements to the national Central Bank (Banque de France). This is the motive of a cross-border transfer; it will be retrieved by the Central Bank to calculate the balance of payment.

For transfer creations, the economic type is not mandatory, except for the AFB320 and EDI91 formats if the applicant or the beneficiary is a non-resident. So you need to set up different economic types, so that they will be available for future selections.

A list of the most used economic types is supplied with the product (see Appendix), but you can complete it to adapt it to your needs.

Feeding Economic Type Table
  1. To access this function, in the Dashboards Tasks pane, select the Payment Elements option in the Setup folder of the Payment module.

The setup page for Receipts Elements is displayed. Click Economic Types.

The list of available Economic Types is displayed.

  1. Click the Automatic Initialization button to view the Selection dialog box.
  2. Choose one of the following selection modes:
  • Click All, if you plan to use all the preset Economic Types. The selection boxes are ticked. You can still deselect types by clicking their selection boxes.
  • If you only want some of the economic types, select them one by one.
  1. Click OK to validate your selection, then confirm, clicking Yes in the dialog box.

The Economic Types page has consequently been modified.

The Create button of the Economic Types contextual menu displays the Economic Types - Create dialog box. Economic Types are imposed by the French Central Bank, and with the list supplied, you may not need to create other type. However, you can customize their descriptions if you find them not clear enough.

Economic Types codes cannot be modified. You need to delete this type and recreate it with a new code.
Creating Economic Types
  1. To access this function, in the Dashboards Tasks pane, select the Payment Elements option in the Setup folder of the Payment module.

The setup page for Receipts Elements is displayed. Click Economic Types.

The list of available Economic Types is displayed.

  1. Click the Create button to view the Economic Types - Create dialog box.

  1. The Code field must be completed. The code consists of maximum three alphanumerical characters.
  2. Enter a description for the type in the Description field. Your description cannot exceed 35 characters.
  3. Once you completed the mandatory fields, click Apply to validate your action and go on with the types creation, or click OK to save the new type and close the dialog box.
Modifying Economic Types
  1. Select the Payment Elements option in the Setup folder of the Payment module. Click Economic Types.
  2. Select from the list the economic type you want to modify.
  3. Click Change. The Economic Types - Modify dialog box is displayed.
  4. Edit the content of the Description field, then click OK to save your action.
Deleting Economic Types
  1. Select the Payment Elements option in the Setup folder of the Payment module. Click Economic Types.
  2. Select from the list the type you want to delete.
  3. Click Delete. The Economic Types - Delete dialog box is displayed.
  4. Confirm your action by clicking Yes in the confirmation dialog box.
