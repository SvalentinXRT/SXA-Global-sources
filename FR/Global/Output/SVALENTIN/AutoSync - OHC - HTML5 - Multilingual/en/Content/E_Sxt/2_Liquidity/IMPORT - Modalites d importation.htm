Setting up Import Terms

Setting up the Transactions Import Terms consists in specifying the total floor of the amounts, i.e. the limit from which transactions with the same budget code, transaction date and currency are imported individually and not associated in a global transaction.

You can define general terms at the level of the file structure. You can also create specific terms for the same structure and for a specific company and/or budget code.

Accessing Setup Page for Import Terms

In the Modules Tasks pane, select the Budget Import Terms option in the Setup folder of the Liquidity module.

The setup page for import terms is displayed with the list of the set up terms in a table. The table shows elements such as the structure code, the target budget sheet and the import mode (add or replace).

The table also displays the specified import terms: the budget code, the company code, the amount and currency of the total floor, as well as the date, currency and description of the totalling rule.

Creating Import Terms
  1. In the setup page for import terms, click the Create button.
    The setup page for import terms opens up.

  1. Enter the basic information of the structure.

View Fields Description

Field Description
Identification area
Structure This field displays the file structure associated with the import terms that you are creating.
Click the button to display the list of the set up file structures. In the contextual window, double-click a structure to select it.
The code and description of the selected structure are displayed in the Structure field.
Budget Code Optional field. It enables you to associate these terms with a specific budget code.
Click the button to display the list of budget codes. In the contextual window, double-click a code to select it.
The code and description of the selected budget code are displayed in the Budget code field.
Company Optional field. It enables you to associate these terms with a specific company.
Click the button to display the list of companies. In the contextual window, double-click a company to select it.
The code and description of the selected company are displayed in the Company field.
Totalling Rules area
The transactions with the same budget code, currency and date can be associated in a single and global transaction.
You can define a total floor i.e. a limit from which transactions are not associated any more, but imported individually.
Total floor Optional field.
To select the currency of the total floor, click the button to display the list of currencies. In the contextual window, double-click a currency to select it.
Then enter the amount for the total floor in the entry field.
If you enable the three options described below, the floor will be applied to the three selected criteria.
Same Date Enable this option so that the total floor is applied to the transactions with the same date.
Same TRX Currency Enable this option so that the total floor is applied to the transactions with the same currency.
Same Description Enable this option so that the total floor is applied to the transactions with the same description.
  1. Click the Default Budget Codes tab.

This page enables you to set up two default budget codes for receipts and disbursements.

The Disbursement field displays the budget code assigned by default to the transactions for which the amount is negative and the Receipt fields displays the budget code and assigned by default to the transactions with a positive amount.

  1. To assign a budget code to the receipts and disbursements, click the button to display the list of budget codes. In the contextual window, double-click a code to select it.
    The code and description of the selected budget code are displayed in the Budget code field.

The Apply button enables you to save your modifications without quitting the current step.

Once you completed the setup of the import terms, click the OK button to validate your entry or Cancel to cancel it.

The setup page for import terms is displayed with the new terms at the bottom of the table.
