Posting and Exporting

The transactions performed during the compensation process are payments.

The posting into Sage XRT Treasury, particularly in Payment and Cash, represents the compensation itself.

The screenshot below illustrates the posting organization: the General area recaps the elements specific to the Netting module (cycle, simulation, etc.), whereas the Participants and Transfer areas recap the elements specific to the Payment module.

If you do not use Cash and Payment products from Sage XRT Treasury, you can still export your validated transactions into your payment system.

Exporting consists in generating a flat file from the database.

The exported data comply with a predefined and fixed structure consisting of fields with fixed lengths and formats.

First of all, the structures of files to be exported must be set up in Netting. You need to know the file structure required by the external system to set up properly this structure.

For more information, refer to the section: Export Structures for Payment Transactions.

Accessing Setup Page for Export tasks

In the Modules Tasks pane, select the Export of Exchange Transactions option in the Import/Export folder of the Netting module.

The page of file export tasks is displayed with the list of the tasks stored in a table.

The table shows the task number and description, the file location and the associated file structure.

  • Reorganization of the lines display in the table, click here
  • Text search in the table, click here
  • Line creation, see the section below
  • Line modification in the table, click here
  • Line deletion in the table, click here
  • Run a background, remote or command line task from Netting interface, refer to the Action Bar section.
Creating and Activating Export Tasks
  1. In the setup page for Export Tasks, click the Create button.
    The first page of the Export wizard opens up.

    A sequential number is automatically created for the export task and displayed in the field You are going to create the task number.
  2. Enter a description for the export task in the Description field.
  3. In the File field, click the button to view the Netting browser and select a file to test.
    A contextual menu appears.
  4. In the browser, select the directory where the file is located.
    The path appears in the Selected Path field and the files of the directory are listed in the table at the bottom of the window.
  5. Double-click the name of the file to select it, or click the button to validate the selection.
    The wizard appears again and you can view the file path in the File field.
  6. In the Structure area, click the button to display the list of the set up structures. In the contextual window, double-click the line to select it.
  7. Click the button Setting up the export options to go on with the setup.
    The setup page for options is displayed.

    If you want:
    • To go to the next step and save the task just as it is, without launching it, click the Validate button
    • To cancel the current setup and redisplay the list of the export tasks, click the Cancel button
    The different tabs of this page enable you to define the filtering criteria if you do not want to export all the FX transactions.
    The definition of these filtering criteria for the transactions export is optional. If you want to export all the FX transactions, click here to go directly to the export step, without going through the steps of the filtering criteria definition.
    The Entities tab enables you to specify filtering criteria on the applicant companies of the FX transactions to export.
  8. In the Select area, enable the following option:
    • All not to apply any filter on the companies,
    • Single to select a single applicant company (selection in the area below),
    • Multiple to select several applicant companies (selection in the area on the right).
  9. In the Select area, if you selected the following option:
    • Single, select a company by clicking the button in the area below, then, in the contextual window, double-click your company
    • Multiple, select several companies as described below
      1. On the right of the Select area, click the Select button.
      2. In the contextual window, enable the options in front of the codes for the companies to select, and click the button at the top of the window.
        The codes and descriptions for the selected companies are displayed in the table of the area.
  10. In the Additional Criteria area, you can add other filtering criteria on the countervalue currency of the transactions, using the button as described below.
  11. Click on the Main Currencies tab.

    This tab enables you to specify filtering criteria on the main currencies of the FX transactions to export.
  12. In the Select area, enable the following option:
    1. All not to apply any filter on the main currencies
    2. Single, to include only the FX transactions which use the selected currency as main currency (in the area below)
    3. Multiple to include the FX transactions which use one of the selected currencies as main currency (in the area on the right)
  13. Go through the same process as in the Entities tab, to select one or several main currencies.
  14. Click on the Period tab.

    This tab enables you to specify filtering criteria on the transaction and delivery dates of the FX transactions to export.
  15. To define a filtering process on the transaction date:
    1. Enable the option TRX Date.
      The fields next to the option are enabled.
    2. In these fileds, specify the start and end date of the period, between which must fall the transaction date of the transactions to export, by going through the following process:
      • In the entry field, select the value to modify and enter a date with the format dd.mm.yyyy.
        - or -
      • Click the button to display the calendar.
        A contextual window displays the calendar.
        View Calendar

        Use the arrows and to select the month, year and day.
        - or -
        To set the start date to today's date, click the button Today.
  16. To specify a filtering process on the delivery date, enable the Delivery Date option, then go through the same process as above, to define the start and end dates of the period, between which must fall the delivery date for the transactions to export.
  17. Click on the Options tab.

    This tab enables you to specify filtering criteria on the back office validation and settlement direction of the transaction.
    It also enables you to add an indicator on the exported transactions. This indicator helps you not to export a same transaction twice.
  18. In the Back-office Validation dropdown list of the Transaction Status area, select a filtering criterion on the back-office validation of transactions:
    • Non B.O. Validated to select the non-BO-validated transactions only
    • B.O. Validated to select the BO-validated transactions only
    • B.O. Validated and Non B.O. Validated to disable any filtering process on the back-office validation
  19. From the Transaction Direction dropdown list, select a filtering criterion for the direction of the transaction:
    • Buy to select the buy transactions only
    • Sell to select the sell transactions only
    • Buy & Sell to disable any filtering process on the transaction direction
  20. In the Mark the exported Netting entries area, enable the Yes option if you want to add an indicator on the exported transactions. This will avoid double export during a later export process.
  21. In the Export Netting Entries area, enable the option:
    • Not exported if you want only the transactions that have not been exported yet, to be exported (the transactions with the export indicator are excluded)
    • Exported if you want only the transactions that have already been exported, to be exported (i.e. the transactions marked with the export indicator)
    • All if you want to cancel any filter on the export indicator
  22. Click on the Transactions tab.

    This tab enables you to specify filtering criteria on the type of the FX transactions to export.
  23. In the Select area, enable the following option:
    1. All not to cancel any filter on transaction types
    2. Single to include only the FX transactions with the selected type (in the area below)
    3. Multiple to include the FX transactions with one of the selected types (in the area on the right)
  24. Go through the same process as in the Entities tab, to select one or several transaction types.
  25. Click on the Advanced tab.

    This tab enables you to specify filtering criteria on the different charcateristics of FX transactions, such as the order number, the external reference, the file number or the portfolio number, the type or the status of the transaction.
  26. In the Number field of the Transaction area, enter a range of order numbers for transactions.
    Important: They are not the complete numbers of transactions, but the numbers of the creation order, they can be found on the right of transactions numbers. The transaction number consists of the company code, the transaction type code and the transaction order number.
  27. In the Link field, enter a range of link numbers for transactions, or select a range using the button as explained previously.
  28. In the Reference field, enter the external reference number for transactions.
  29. In the File field, enter the file number of the transactions, or select the file using the button .
  30. In the Portfolio field, enter the portfolio number of transactions, or select the portfolio using the button.
  31. In the Nature area, enable the options corresponding to the transaction statuses that you want to view:
    • Initial to select the initial transactions (manual creations)
    • Exercising to select the forward exchange transactions created after the early exercise
    • Deferring to select the forward exchange transactions created after a deferment,
    • Swap to select the swap transactions
    • Exercised to select the forward exchange transactions which have been early exercised
    • Deferred to select the forward exchange transactions which have been deferred
  32. In the Status area, select the option:
    • Opened to select the non-settled transactions only
    • Closed to select the settled transactions only
      If you want:
    • To save the task just as it is, without launching it, click the Validate button.
    • To cancel the current setup and redisplay the list of the export tasks, click the Cancel button
  33. Click the Export button to initiate the export task.
    The export is launched and a new page is displayed showing the task progress.
  34. If you want to stop the task, click the Cancel button at the bottom of the page.
  35. Once the task is completed, click the Close button at the bottom of the page.
    The previous page is redisplayed.
  36. Click the Validate button to save the task.
    The setup page for export tasks re-appears with the new task at the bottom of the list.
