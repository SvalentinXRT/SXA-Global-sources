Execution

Before importing, make sure you completed the specific setup required to launch the import phase, i.e.:

  • The structure for the file to import (we recommend you test it before)
  • The import terms

The import of Budget Transactions consists of two processes which can be performed simultaneously or successively, depending on your needs:

  • Creating and saving the import task
  • Executing the task

See the previous topics for more information on these processes.

Accessing Setup Page for import tasks

In the Modules Tasks pane, select the Import Budget Transactions option in the Import/Export folder of Liquidity.

The tasks page for file import is displayed with the list of the import tasks saved in a table. The table shows the task description, the file location and the associated file structure.

For more information on:

  • how to reorganize the lines display in the table, click here
  • how to search text in the table, click here,
  • how to create lines, see the topic below,
  • Modifying the structure and equivalences in the table, click Modify the structure or Modify equivalences (you can access these pages in creation mode, for which the process is described in the topic below),
  • how to delete lines in the table, click here.
Creating and Executing Import Tasks
  1. In the setup page for import tasks, click the Create button.
    The first page of the import wizard opens up.

A sequential number is automatically created for the import task and displayed in the field You are going to create the task number.

  1. Enter a description for the import task in the Description field.
  2. In the File field, click the button to view the Liquidity browser and select a file to test.

A contextual menu appears.

  1. In the browser, select the directory where the file is located.
    The path appears in the Selected Path field and the files of the directory are listed in the table at the bottom of the window.
  2. Double-click the name of the file to select it, or click the button to validate the selection.

The wizard appears again and you can view the file path in the File field.

  1. In the Structure area, click the button to display the list of set structures. In the contextual window, double-click the line to select it.
  2. Click the button  Setting up the import options to go on with the setup.

The setup page for Options is displayed.

It enables you to define the following elements, among others:

  • whether imported data will be crated in the Operational Budget or in the Treasury Budget,
  • whether, for this import period, the transactions already included into the budget are totally kept, overwritten or replaced by the imported transactions under certain conditions (same budget code and same transaction date).
  1. If you want to:
  • go to the next step and save the task just as it is, without launching it, click the Validate button,
  • cancel the current setup and redisplay the list of the export tasks, click the Cancel button.
Field Description

Delete File after Import

Enable this option if you do not want to keep the file when the import is completed.
Status of entries in the file area
Consider data in file as Budget entries

Enable this option to define the created transactions as Budget transactions.

The Origin field of the created transactions is set to "Budget".
Consider data in file as Treasury entries

Enable this option to define the created transactions as actuals.

The Origin field of the created transactions is set to "Treasury".
Import Mode area
Add Enable this option to add the transactions of the file to those already saved in the database.
Remember that if a transaction exists in the database AND in the file for the same date, the same budget code, the same company and with a same amount, this transaction will be posted twice.
Replace Then Add Enable this option to replace in the database the entered amounts by those in the file, when these transactions have the same date, the same budget code and the same company (whatever the amount).
Remember that this mode enables you to overwrite only the entered amounts. The amounts with formula-based calculation are kept.
Replace All Enable this option to overwrite all the transactions included in a specified (by you) time span and to create the transactions from the file.
Destination area
Operational Budget Sheet (OBS) Enable this option so that the transactions are imported into the Operational Budget.
Treasury Budget Sheet (TBS) Enable this option so that the transactions are imported in the Treasury Budget.

Dates area

This area is editable only if you enabled the Replace all option.

Start Date Enter a start date for the import period in the entry field or use the calendar. Click here for more information on how to use the calendar.
Exact Date

Use this option if you know the exact date for the end of the import period.

Then, define the end date in the End Date field.
End Date

This field is editable if you selected the Exact Date option.

Enter an end date for the import period in the entry field or use the calendar. Click here for more information on how to use the calendar.
Rolling Date

Use this option for a formula-based calculation of the period end date.

Then, enter the formula in the entry field next to this option.
Entry Field

This field is editable if you selected the Rolling Date option.

Its default value is "Date".

Enter a calculation formula in this field, as described in the Dates and Periods topic.
Exercise Impact area
The setup of the following area is critical to the management of one exercise per currency.
Impact single exercise with currency If you manage one exercise per currency and that you want to import transactions into a single exercise or if all currencies are managed in the same exercise, select this option and the currency of the target exercise using the button .
Impact all possible exercises If you manage one exercise per currency and that you want to import transactions into all the available exercises, select this option.
In this case, the exercise currency is mandatory information, crucial to the import and you must set it up as a mandatory field in the import structure. For more information on setting up this option, click here.
  1. Click the Import button to launch the import task.

If you want to:

  • save the task just as it is, without launching it, click the Validate button,
  • cancel the current setup and redisplay the list of the export tasks, click the Cancel button.

The import is launched and a new page is displayed showing the task progress.

The Totalized field specifies the number of transactions that have been totalized according to the criteria defined in the Total floor area in the setup page for import terms.
  1. If you want to stop the task, click the Cancel button at the bottom of the page.
  2. Once the task is completed, pay attention to the number of rejects, then click the Close button at the bottom of the page.

The previous page is redisplayed.

  1. Click the  Validate button to validate and save the task.

The setup page for import tasks is displayed again with the new task at the bottom of the list.

If the execution of the task generated rejects, see the Rejects topic.

You can view the transactions that have been actually imported in your Budget Sheet.

For more information on how to use Budget Sheets, see the Budget Sheet Entry and Analysis topic.

Launching an import task by command line

You can relaunch, by command prompt, an existing import task.

  1. To view the command line to use, click a line to select the corresponding task on the import task page, then click the Command Line button in the action bar.

The Command line dialog box appears and shows the command and all the required settings: the path of the executable file, the database, the username and password, the task ID number and the ouput log file.

  1. Click the Copy to Clipboard button, then the Close button.
  2. In a text editor, paste the text and replace the following parameters:
  • your.login by your username,
  • your.login by the password associated with your username,
  • log.txt by the name you chose for the output log file.

Do not change the path of the executable file and the task ID number.

  1. Launch the command line.
  2. In the command line, right-click and in the contextual menu, click the Paste option, then validate with the Enter key.
