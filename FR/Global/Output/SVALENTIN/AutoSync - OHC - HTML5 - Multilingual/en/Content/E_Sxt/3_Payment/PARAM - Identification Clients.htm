Client Identification

Client Identifications are specific to some transactions. This optional element can be required by the branches using the AFB230 format. The ID number is supplied by the branch.

The Client Identification command enables you to enter once and for all the code that the branch needs when it receives your remittance files.

Setting up Client Identification
  1. To access this function, in the Dashboards Tasks pane, select the Payment Elements option in the Setup folder of the Payment module.

The setup page for Receipts Elements is displayed. Click Client Identifications.

The list of the already set up identifications, is displayed. You can delete or modify an existing identification, or create another one. To create a client identification, click Create.

  1. Enter the code of the branch that receives the AFB320-formatted remittances, in the Applicant fields, or select it from the list by clicking the button .
  2. Repeat this step for the applicant code.
  3. Enter a client identification with maximum 16 alphanumerical characters.
  4. Click Apply to validate your action and go on with your creations, or click OK to save your new identification and go back to the list of the already created IDs.
Modifying Client Identification
  1. Select the Payment Elements option in the Setup folder of the Payment module. Click Client Identifications.
  2. Select from the list the element you want to modify.
  3. Click Change. The detailed page for the client identification is displayed.
  4. Edit the content of the Client Identification field, then click OK to save your action.
Deleting Client Identification
  1. Select the Payment Elements option in the Setup folder of the Payment module. Click Client Identifications.
  2. Select from the list the element you want to delete.
  3. Click Delete to delete the selected client ID.
  4. Confirm your action by clicking Yes in the confirmation dialog box.
