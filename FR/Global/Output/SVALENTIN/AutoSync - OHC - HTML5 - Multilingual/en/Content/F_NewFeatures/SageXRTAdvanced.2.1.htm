Sage XRT Advanced 2.1

IBAN Validation

This function enables the validation of IBAN account number during import or manual input.

86 formats (countries) can be processed: data structures, number and length validation.

A calculation option for the account ID key is also available.

For manual input, contextual help is offered to comply with the target format.

Sub-Accounts

This function enables the management of several account IDs in the same currency on the same bank account. All the entries for these accounts are integrated and grouped on the new bank account.

You can add as many account IDs as necessary into the bank account setup, in Identifiers tab. One of the available account IDs must be defined as the main Acct. ID.

Calendars

In the Setup module, the Branch Calendar menu enables the definition of the annual bank holidays and weekly closed days for a branch.

These days are applied to all the accounts held by these entities.

With these calendars, the value date applied to each transaction by the bank can be calculated.

Several calendars are required when those days vary according to the bank branches.

The setup process consists of two steps:

  • creating the calendars
  • linking them to branches

In the Setup menu, click Create on the right and complete the form.

Batch Import

In the Administration module, a new function to import files by batch is available in Import/Export - Incoming File.

The Create by Batch action enables the selection of several files. Once validated, the action generates as many import tasks as imported files. Select these tasks and launch the import process.

Selected Edition

For every module and report, you can now refine reports by selecting specific elements.

These elements simply need to be selected (check their boxes) before the edition process.

For example, the Reconciliation Report by Account has been focused on the three selected accounts.

Balances and Dates Consistency

For Reconciliation and Cash modules, both the consistency between final and initial balances and the consistency between final and initial dates are automatically checked during the import of account statements.

The following conditions must be met:

  • the account statement must include an opening and a closing balance
  • for each account statement, the opening balance received should be equal to the amount of the last closing balance imported
  • for each account statement, the date of the opening balance received should be the same as for the last closing balance imported.

If there is any inconsistency, the statement is rejected.

Bank Delegations Module

Letter Generation

During the definition of the letter generation rules, several accounts linked to different companies may be selected. A single letter is generated for all the accounts and sent to the main company.

This setup is available in the Accounts section, of the Settings tab.

More Information

During the letter generation, the companies for each account can be identified.

Reconciliation Module

Reconciliation Dates Storage

After each manual and automatic reconciliation, the process date is stored. It is available in the reconciliation filters.

G/L Number for More than 1 Company

The new field G/L Number is available in the G/L account setup. This field must be specified and can be common to several companies.

Through this new feature, the combo Identical G/L Number - Company is now possible. This combination is linked to a bank account.

E.g.:

  • (Company A, 512001) linked to (CMPTX, Company A)
  • (Company B, 512001) linked to (CMPTX, Company B)

In the import structure, the company field is mandatory for identification purposes.
