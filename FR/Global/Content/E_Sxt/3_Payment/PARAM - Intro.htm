Paramétrage

Le paramétrage doit être effectué pour les éléments ci-dessous avant la gestion des demandes de paiement et virements de trésorerie.

  • Droits d'accès des utilisateurs aux fonctions

Ce paramétrage permet de cloisonner les droits d'accès à chaque fonction du module par type d'utilisateur.

  • Les utilisateurs front office créent les transactions.
  • Les utilisateurs back office valident les transactions.

Pour plus d'informations sur la gestion des droits d'accès, consultez la documentation du module .

  • Référentiel

Le paramétrage du référentiel concerne les données stables : entités, banques, comptes bancaires, etc.

Cette configuration s'effectue dans le module .

  • Cours de change

Cette configuration s'effectue dans le module .

Les données de référentiel et de change à paramétrer sont répertoriées dans la sectionrubrique suivante.

  • Configuration spécifique

La configuration spécifique au module concerne, entre autres, la personnalisation des profils et règles de constitution, la personnalisation des modèles de règlement et la structure des fichiers échangés avec les systèmes externes.

La configuration spécifique au module est détaillée dans les sectionsrubriques suivantes.
