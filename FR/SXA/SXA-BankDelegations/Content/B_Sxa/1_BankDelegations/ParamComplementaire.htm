Paramétrage complémentaire

Avant de pouvoir commencer à utiliser le module les données suivantes doivent également être paramétrées :

Type d'identifiant

Par exemple, le code Insee SIREN3

 Catégorie de Compte

La liste suivante est livrée par défaut :

  • 0 = indéfini
  • 1 = Compte d'épargne
  • 2 = Compte courant
  • 3 = Compte titre
  • 4 = Compte devise
  • 5 = Autre

Format des Comptes

Par exemple, DEF-FR définit par défaut le format de compte français dans l'application.

 Contacts Bancaires

Les contacts bancaires sont associés aux agences bancaires. Il est possible d'associer plusieurs contacts à la même agence, à condition de définir un contact principal. Ces contacts sont utilisés dans les lettres.

 Collaborateurs

Il est possible de désigner les employés créés en tant que délégant et / ou délégataire et également d'y joindre des documents supplémentaires.

Dans l'onglet Extension, après avoir coché la case Utiliser Collaborateur pour Extension, une case à cocher permet de spécifier si le collaborateur est un délégant ou un délégataire.

 Groupe de Collaborateurs

Il est possible de regrouper plusieurs collaborateurs pour former un groupe. Un même collaborateur peut appartenir à différents groupes de collaborateurs.

 Sociétés

Désignation d'un mandant.

 Type d'opération

Les types d'opération sont paramétrés par l'utilisateur final. Le libellé du type d'opération saisi lors de la création est utilisé dans les lettres.

Autres exemples de libellés :

  • Signer tous les chèques
  • Signer les autorisations
  • Émission des crédits documentaires
  • Signer des ordres de virements, etc.
  • Type de signature
Type de signature

Les types de signature sont paramétrés par l'utilisateur final. Le libellé du type de signature saisi lors de la création est utilisé dans les lettres.

Autres exemples de libellés :

  • Agissant séparément
  • Agissant conjointement
  • 1er signataire
  • 2ème signataire, etc.
